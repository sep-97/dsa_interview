# فصل ۱: تحلیل الگوریتم

> ایده خیلی ساده: می‌خواهیم بفهمیم اگر تعداد داده‌ها زیاد شود، برنامه چقدر کار بیشتر انجام می‌دهد و چقدر حافظه بیشتر می‌خواهد.

## مثال روزمره

فرض کن دنبال اسم یک نفر در دفتر تلفن هستی. اگر از اول دفتر یکی‌یکی بگردی، شاید مجبور شوی همه صفحه‌ها را ببینی.

اما اگر دفتر مرتب باشد، می‌توانی از وسط شروع کنی و هر بار نصف دفتر را کنار بگذاری. هر دو روش جواب می‌دهند، ولی سرعتشان فرق دارد.

تحلیل الگوریتم یعنی همین فرق را قبل از خراب شدن برنامه بفهمیم.

## حالا در برنامه‌نویسی یعنی چه؟

اندازه ورودی را معمولا با n نشان می‌دهیم. مثلا اگر آرایه ۱۰۰۰ عدد داشته باشد، n برابر ۱۰۰۰ است.

Big O به ما می‌گوید با بزرگ شدن n، تعداد عملیات چطور رشد می‌کند. O(1) یعنی تقریبا ثابت، O(n) یعنی متناسب با تعداد داده، O(n²) یعنی خیلی سریع‌تر و خطرناک‌تر رشد می‌کند.

Space Complexity هم می‌گوید الگوریتم چقدر حافظه اضافی مصرف می‌کند.

## مثال C#

</div>

```csharp
public static bool Contains(int[] numbers, int target)
{
    foreach (var number in numbers)
    {
        if (number == target)
            return true;
    }

    return false;
}

// Time: O(n) چون شاید همه عددها را ببینیم
// Space: O(1) چون فقط چند متغیر داریم
```

<div dir="rtl" align="right">

## چیزهایی که باید از این فصل یاد بگیری

- Big O زمان دقیق اجرا نیست؛ مدل رشد است.
- دو راه‌حل درست می‌توانند در داده بزرگ رفتار کاملا متفاوت داشته باشند.
- در مصاحبه باید هم زمان و هم حافظه را توضیح بدهی.

## پیچیدگی‌ها و مقایسه‌ها

| Big O | مثال ساده | حس آن |
| --- | --- | --- |
| O(1) | برداشتن چیزی از کمد شماره‌دار | مستقیم می‌روی سراغش |
| O(log n) | حدس عدد با نصف کردن بازه | هر قدم نصف گزینه‌ها حذف می‌شود |
| O(n) | گشتن در صف | هر نفر شاید بررسی شود |
| O(n²) | مقایسه همه با همه | با زیاد شدن داده خیلی گران می‌شود |

## اشتباهات رایج

- فکر کنیم چون کد روی داده کم سریع است، همیشه سریع می‌ماند.
- حافظه را حساب نکنیم.
- دو حلقه پشت سر هم را با دو حلقه تو در تو اشتباه بگیریم.

## در مصاحبه چطور بگویم؟

> در بدترین حالت باید همه عناصر را ببینم، پس زمان O(n) است. چون ساختار داده جدیدی نمی‌سازم و فقط چند متغیر دارم، حافظه O(1) است.

## تمرین

1. برای یک حلقه ساده، دو حلقه پشت سر هم، و دو حلقه تو در تو Big O را بنویس.
2. مثالی بزن که با مصرف حافظه بیشتر، زمان کمتر شود.
3. Binary Search را با دفتر تلفن برای خودت توضیح بده.

## پاسخ تمرین‌ها

1. یک حلقه ساده روی n آیتم برابر O(n) است. دو حلقه پشت سر هم O(n+n) یعنی O(n) است. دو حلقه تو در تو که هر دو تا n می‌روند O(n*n) یعنی O(n²) است.
2. مثال: Two Sum. با دو حلقه حافظه O(1) داریم ولی زمان O(n²) می‌شود. با HashSet یا Dictionary حافظه O(n) مصرف می‌کنیم، اما زمان به O(n) می‌رسد.
3. در دفتر تلفن مرتب، اگر هر بار وسط را نگاه کنیم و نصف اشتباه را کنار بگذاریم، تعداد قدم‌ها O(log n) می‌شود. شرطش مرتب بودن داده است.

# فصل ۲: آرایه و List

> ایده خیلی ساده: Array مثل کمدهای شماره‌دار پشت سر هم است. اگر شماره خانه را بدانی، مستقیم می‌روی سراغش.

## مثال روزمره

فرض کن یک ساختمان ۱۰ واحدی داری و واحدها پشت سر هم شماره‌گذاری شده‌اند. اگر بگویی واحد ۷، لازم نیست از واحد ۱ شروع کنیم.

مستقیم می‌رویم سراغ واحد ۷. Array هم همین حس را دارد: با index مستقیم به خانه مورد نظر می‌رسی.

اما اگر بخواهی وسط ساختمان یک واحد جدید اضافه کنی، همه واحدهای بعدی باید جابه‌جا شوند. این بخش سخت array است.

## حالا در برنامه‌نویسی یعنی چه؟

Array در حافظه پشت سر هم ذخیره می‌شود. به همین دلیل دسترسی با index برابر O(1) است.

در .NET، نوع List<T> در پشت صحنه شبیه یک آرایه پویا یا dynamic array کار می‌کند. یعنی یک array داخلی دارد و وقتی جا کم بیاورد، array بزرگ‌تری می‌سازد و داده‌ها را کپی می‌کند.

Add به انتهای List معمولا سریع است، ولی Insert یا Remove وسط لیست می‌تواند O(n) باشد.

## مثال C#

</div>

```csharp
var names = new List<string>();
names.Add("Ali");
names.Add("Sara");
names.Add("Reza");

Console.WriteLine(names[1]); // Sara

// دسترسی با index: O(1)
// اضافه کردن به انتها: معمولا amortized O(1)
// حذف از وسط: O(n)
```

<div dir="rtl" align="right">

## چیزهایی که باید از این فصل یاد بگیری

- Array برای دسترسی با index عالی است.
- List<T> وقتی تعداد آیتم‌ها تغییر می‌کند راحت‌تر از array خام است.
- حذف و اضافه وسط List گران است چون عناصر باید جابه‌جا شوند.

## پیچیدگی‌ها و مقایسه‌ها

| عملیات | پیچیدگی | چرا؟ |
| --- | --- | --- |
| Access by index | O(1) | آدرس مستقیم حساب می‌شود |
| Add آخر List | Amortized O(1) | گاهی resize رخ می‌دهد |
| Insert وسط | O(n) | باید عناصر بعدی جابه‌جا شوند |
| Remove اول/وسط | O(n) | باید جای خالی پر شود |
| Search نامرتب | O(n) | شاید همه عناصر دیده شوند |

## اشتباهات رایج

- از List<T> مثل Queue استفاده کنیم و مدام RemoveAt(0) بزنیم.
- Capacity و Count را یکی بدانیم.
- برای حجم خیلی بالا، هزینه resize و allocation را نادیده بگیریم.

## در مصاحبه چطور بگویم؟

> Array برای index access سریع است، چون خانه‌ها پشت سر هم‌اند. اما درج یا حذف وسط گران است، چون عناصر بعدی باید shift شوند.

## تمرین

1. یک List بساز و چند Add، Insert و Remove انجام بده.
2. توضیح بده چرا names[2] سریع است ولی RemoveAt(0) کندتر است.
3. فرق Count و Capacity را با یک مثال بنویس.

## پاسخ تمرین‌ها

1. بعد از Add، مقدار به انتهای List اضافه می‌شود. Insert وسط باعث می‌شود عناصر بعدی یک خانه جلو بروند. Remove وسط هم باعث shift شدن عناصر بعدی به عقب می‌شود.
2. names[2] سریع است چون index مستقیم داریم و آدرس خانه حساب می‌شود. RemoveAt(0) کندتر است چون همه عناصر بعد از خانه صفر باید یک خانه به عقب منتقل شوند.
3. Count تعداد آیتم‌های واقعی داخل List است. Capacity اندازه array داخلی است. ممکن است Count برابر 3 باشد ولی Capacity مثلا 4 یا 8 باشد.

# فصل ۳: Linked List

> ایده خیلی ساده: Linked List مثل قطار است. هر واگن می‌داند واگن بعدی کجاست، ولی واگن‌ها لازم نیست در یک جای پشت سر هم باشند.

## مثال روزمره

در array خانه‌ها کنار هم هستند. اما در linked list هر خانه، آدرس خانه بعدی را نگه می‌دارد.

مثل زنجیر: برای رسیدن به حلقه پنجم باید از اول زنجیر حرکت کنی. نمی‌توانی مستقیم بپری روی حلقه پنجم، مگر از قبل آدرسش را داشته باشی.

مزیتش این است که اگر بخواهی یک حلقه وسط زنجیر اضافه یا حذف کنی، فقط چند اتصال را عوض می‌کنی.

## حالا در برنامه‌نویسی یعنی چه؟

هر node معمولا value و next دارد. در doubly linked list، prev هم داریم.

دسترسی به index خاص O(n) است، چون باید از head جلو برویم.

اما اگر خود node را داشته باشیم، حذف یا اضافه کنار آن می‌تواند O(1) باشد.

## مثال C#

</div>

```csharp
public sealed class Node<T>
{
    public T Value { get; }
    public Node<T>? Next { get; set; }

    public Node(T value) => Value = value;
}

var first = new Node<int>(10);
var second = new Node<int>(20);
first.Next = second;
```

<div dir="rtl" align="right">

## چیزهایی که باید از این فصل یاد بگیری

- Linked List برای دسترسی تصادفی خوب نیست.
- برای جابه‌جایی اتصال‌ها و حذف/درج node معلوم مفید است.
- LRU Cache یکی از کاربردهای مهم doubly linked list است.

## پیچیدگی‌ها و مقایسه‌ها

| عملیات | پیچیدگی | توضیح |
| --- | --- | --- |
| Get index | O(n) | باید از head حرکت کنیم |
| Search | O(n) | شاید همه nodeها دیده شوند |
| Insert بعد از node معلوم | O(1) | فقط اتصال‌ها عوض می‌شوند |
| Remove node معلوم در doubly | O(1) | prev و next داریم |

## اشتباهات رایج

- فکر کنیم LinkedList همیشه از List سریع‌تر است.
- head و tail را بعد از حذف درست update نکنیم.
- حالت لیست خالی یا تک‌عضوی را فراموش کنیم.

## در مصاحبه چطور بگویم؟

> Linked List برای index access کند است، اما اگر node را داشته باشیم، تغییر اتصال‌ها سریع است. به همین دلیل برای LRU با Dictionary ترکیب می‌شود.

## تمرین

1. یک linked list سه nodeای روی کاغذ بکش.
2. Reverse linked list را با سه pointer توضیح بده.
3. Cycle detection را با slow و fast pointer تمرین کن.

## پاسخ تمرین‌ها

1. در یک linked list سه nodeای، head به node اول اشاره می‌کند، node اول به دوم، و node دوم به سوم. node سوم معمولا Next برابر null دارد.
2. برای reverse، سه اشاره‌گر previous، current و next نگه می‌داریم. هر بار current.Next را به previous تغییر می‌دهیم و جلو می‌رویم.
3. در cycle detection، slow یک قدم و fast دو قدم حرکت می‌کند. اگر به هم برسند cycle داریم؛ اگر fast به null برسد cycle نداریم.

# فصل ۴: Stack و Queue

> ایده خیلی ساده: Stack مثل دسته بشقاب است؛ آخرین بشقاب اول برداشته می‌شود. Queue مثل صف نانوایی است؛ اولین نفر اول نان می‌گیرد.

## مثال روزمره

وقتی چند بشقاب را روی هم می‌گذاری، آخرین بشقاب بالاست و اول برداشته می‌شود. این می‌شود Stack.

اما در صف نانوایی، کسی که زودتر آمده باید زودتر سرویس بگیرد. این می‌شود Queue.

خیلی از الگوریتم‌ها فقط با فهم همین نظم حل می‌شوند.

## حالا در برنامه‌نویسی یعنی چه؟

Stack مدل LIFO دارد: Last In, First Out. برای undo، call stack، DFS و بررسی پرانتزها مفید است.

Queue مدل FIFO دارد: First In, First Out. برای BFS، صف jobها و پردازش پیام‌ها مفید است.

در .NET از Stack<T> و Queue<T> استفاده می‌کنیم.

## مثال C#

</div>

```csharp
public static bool IsValidParentheses(string text)
{
    var stack = new Stack<char>();

    foreach (var ch in text)
    {
        if (ch == '(') stack.Push(ch);
        if (ch == ')')
        {
            if (stack.Count == 0) return false;
            stack.Pop();
        }
    }

    return stack.Count == 0;
}
```

<div dir="rtl" align="right">

## چیزهایی که باید از این فصل یاد بگیری

- Stack برای وقتی خوب است که آخرین چیز باید اول بررسی شود.
- Queue برای پردازش به ترتیب ورود مناسب است.
- DFS معمولا با Stack و BFS معمولا با Queue فکر می‌شود.

## پیچیدگی‌ها و مقایسه‌ها

| ساختار | مثال روزمره | کاربرد برنامه‌نویسی |
| --- | --- | --- |
| Stack | دسته بشقاب | Undo، DFS، پرانتزها |
| Queue | صف نانوایی | BFS، job queue |
| Deque | صف دوطرفه | Sliding window پیشرفته |

## اشتباهات رایج

- قبل از Pop یا Dequeue خالی بودن را چک نکنیم.
- برای Queue از List.RemoveAt(0) استفاده کنیم.
- نظم LIFO و FIFO را قاطی کنیم.

## در مصاحبه چطور بگویم؟

> اگر مسئله بگوید آخرین چیز باید اول بررسی شود، به Stack فکر می‌کنم. اگر پردازش باید به ترتیب ورود باشد، Queue انتخاب طبیعی است.

## تمرین

1. Valid Parentheses را برای ()() و (() تست کن.
2. Queue را با دو Stack پیاده‌سازی کن.
3. BFS ساده روی یک درخت کوچک را با Queue اجرا کن.

## پاسخ تمرین‌ها

1. برای ()() ابتدا '(' push می‌شود، با ')' pop می‌شود، و همین دوباره تکرار می‌شود؛ در پایان stack خالی است، پس معتبر است.
2. برای Queue با دو Stack، enqueue را در stack اول می‌ریزیم. هنگام dequeue اگر stack دوم خالی بود، همه را از stack اول به stack دوم منتقل می‌کنیم تا ترتیب برعکس و FIFO شود.
3. در BFS، node شروع وارد Queue می‌شود. هر بار یک node خارج می‌کنیم، همسایه‌های دیده‌نشده‌اش را وارد Queue می‌کنیم، و همین تا پایان ادامه دارد.

# فصل ۵: جستجو

> ایده خیلی ساده: جستجو یعنی چیزی را پیدا کنیم. سوال مهم این است: داده مرتب است یا نه؟

## مثال روزمره

اگر یک کشوی شلوغ داری و دنبال کلید می‌گردی، شاید مجبور شوی همه چیز را یکی‌یکی نگاه کنی.

اما اگر کشو مرتب باشد و هر چیز جای مشخص داشته باشد، خیلی سریع‌تر پیدا می‌کنی.

در الگوریتم‌ها هم مرتب بودن یا نبودن داده، نوع جستجو را عوض می‌کند.

## حالا در برنامه‌نویسی یعنی چه؟

Linear Search روی هر داده‌ای کار می‌کند، ولی O(n) است.

Binary Search فقط وقتی کار می‌کند که داده مرتب باشد یا بتوانیم جواب‌ها را به دو بخش درست/غلط تقسیم کنیم.

Hash lookup هم راه دیگری است: با حافظه بیشتر، lookup میانگین O(1) می‌گیریم.

## مثال C#

</div>

```csharp
public static int BinarySearch(int[] sorted, int target)
{
    int left = 0;
    int right = sorted.Length - 1;

    while (left <= right)
    {
        int mid = left + (right - left) / 2;
        if (sorted[mid] == target) return mid;
        if (sorted[mid] < target) left = mid + 1;
        else right = mid - 1;
    }

    return -1;
}
```

## چیزهایی که باید از این فصل یاد بگیری

- اگر داده مرتب نیست، Binary Search مجاز نیست.
- Binary Search هر بار نصف گزینه‌ها را حذف می‌کند.
- گاهی به جای جستجوی value، روی answer search می‌کنیم.

## پیچیدگی‌ها و مقایسه‌ها

| روش | شرط | زمان |
| --- | --- | --- |
| Linear Search | هیچ شرطی ندارد | O(n) |
| Binary Search | مرتب یا monotonic | O(log n) |
| Hash Lookup | ساختار hash | میانگین O(1) |

## اشتباهات رایج

- Binary Search را روی array نامرتب اجرا کنیم.
- مرز left و right را اشتباه update کنیم.
- mid را به شکلی حساب کنیم که overflow بدهد.

## در مصاحبه چطور بگویم؟

> اگر داده مرتب باشد، می‌توانم با Binary Search هر بار نصف گزینه‌ها را حذف کنم؛ پس زمان O(log n) است.

## تمرین

1. Linear Search و Binary Search را روی یک array کوچک دستی اجرا کن.
2. First occurrence را با Binary Search پیدا کن.
3. Search Insert Position را حل کن.

## پاسخ تمرین‌ها

1. Linear Search روی array کوچک همه عناصر را از اول بررسی می‌کند. Binary Search روی array مرتب از وسط شروع می‌کند و هر بار نصف بازه را حذف می‌کند.
2. برای First Occurrence وقتی target پیدا شد، جواب را ذخیره می‌کنیم ولی right را به mid-1 می‌بریم تا ببینیم نمونه زودتری هم هست یا نه.
3. Search Insert Position همان lower_bound است: اولین index که مقدارش بزرگ‌تر یا مساوی target باشد. اگر نبود، جواب انتهای array است.

# فصل ۶: مرتب‌سازی

> ایده خیلی ساده: مرتب‌سازی یعنی داده‌ها را به ترتیبی بچینیم که بعدا راحت‌تر با آن‌ها کار کنیم.

## مثال روزمره

فرض کن کتابخانه‌ای داری که کتاب‌ها بی‌نظم روی زمین ریخته‌اند. پیدا کردن یک کتاب سخت است.

اگر کتاب‌ها را بر اساس نام یا موضوع مرتب کنی، بعدا پیدا کردن و مقایسه کردن خیلی راحت‌تر می‌شود.

مرتب‌سازی گاهی خودش هزینه دارد، اما بعد از آن مسئله‌های دیگر ساده‌تر می‌شوند.

## حالا در برنامه‌نویسی یعنی چه؟

Bubble و Insertion برای یادگیری خوب‌اند، ولی برای داده بزرگ معمولا کندند.

Merge Sort قابل پیش‌بینی و O(n log n) است، ولی حافظه اضافی می‌خواهد.

Quick Sort در عمل سریع است، اما pivot بد می‌تواند آن را O(n²) کند.

## مثال C#

</div>

```csharp
public static void BubbleSort(int[] arr)
{
    for (int i = 0; i < arr.Length; i++)
    {
        for (int j = 0; j < arr.Length - 1 - i; j++)
        {
            if (arr[j] > arr[j + 1])
                (arr[j], arr[j + 1]) = (arr[j + 1], arr[j]);
        }
    }
}

// آموزشی است؛ برای داده بزرگ مناسب نیست.
// Time: O(n²)
```

<div dir="rtl" align="right">

## چیزهایی که باید از این فصل یاد بگیری

- Sort کردن هزینه دارد؛ رایگان نیست.
- بسیاری از sortهای خوب O(n log n) هستند.
- بعد از sort، two pointers و binary search ممکن می‌شوند.

## پیچیدگی‌ها و مقایسه‌ها

| Sort | زمان معمول | نکته |
| --- | --- | --- |
| Bubble | O(n²) | فقط آموزشی |
| Insertion | O(n²) | برای داده کوچک/تقریبا مرتب خوب |
| Merge | O(n log n) | حافظه O(n) |
| Quick | میانگین O(n log n) | بدترین O(n²) |
| Heap | O(n log n) | حافظه کم‌تر، ولی همیشه ساده‌ترین نیست |

## اشتباهات رایج

- برای هر مسئله اول sort کنیم بدون حساب کردن هزینه.
- Stable sort را نادیده بگیریم.
- Quick Sort با pivot بد را فراموش کنیم.

## در مصاحبه چطور بگویم؟

> اگر داده را sort کنم، هزینه O(n log n) می‌پردازم، اما بعد ممکن است مسئله با two pointers یا binary search ساده‌تر شود.

## تمرین

1. Bubble Sort را با ۵ عدد دستی اجرا کن.
2. Merge دو array مرتب را بدون sort مجدد بنویس.
3. توضیح بده stable sort یعنی چه.

## پاسخ تمرین‌ها

1. در Bubble Sort، هر دور عناصر کناری مقایسه و در صورت نیاز swap می‌شوند. بعد از هر دور، بزرگ‌ترین مقدار باقی‌مانده به انتهای بخش نامرتب می‌رود.
2. برای merge دو array مرتب، دو pointer داریم. هر بار مقدار کوچک‌تر را وارد خروجی می‌کنیم و همان pointer را جلو می‌بریم. زمان O(n+m) است.
3. Stable sort یعنی اگر دو آیتم کلید برابر داشته باشند، ترتیب نسبی قبلی‌شان حفظ شود. مثلا در مرتب‌سازی دانشجوها بر اساس نمره، ترتیب قبلی هم‌نمره‌ها حفظ می‌شود.

# فصل ۷: Recursion

> ایده خیلی ساده: Recursion یعنی تابع برای حل مسئله، نسخه کوچک‌تر همان مسئله را حل کند.

## مثال روزمره

فرض کن می‌خواهی تعداد جعبه‌های داخل یک جعبه بزرگ را بشماری. هر جعبه ممکن است خودش چند جعبه دیگر داشته باشد.

می‌گویی: تعداد این جعبه = ۱ + تعداد جعبه‌های داخلش. برای هر جعبه داخلی هم همین قانون را اجرا می‌کنی.

این همان فکر recursive است.

## حالا در برنامه‌نویسی یعنی چه؟

هر recursion باید base case داشته باشد؛ یعنی جایی که دیگر تابع خودش را صدا نزند.

هر فراخوانی روی call stack می‌رود. پس recursion فقط زمان مصرف نمی‌کند، حافظه stack هم مصرف می‌کند.

Treeها، graphها، merge sort و backtracking با recursion خیلی طبیعی توضیح داده می‌شوند.

## مثال C#

</div>

```csharp
public static int Factorial(int n)
{
    if (n <= 1)
        return 1; // base case

    return n * Factorial(n - 1);
}

// Factorial(5) = 5 * Factorial(4)
// Time: O(n)
// Space: O(n)
```

<div dir="rtl" align="right">

## چیزهایی که باید از این فصل یاد بگیری

- Base case جلوی بی‌نهایت شدن را می‌گیرد.
- هر recursion حافظه call stack دارد.
- اگر مسئله کوچک‌تر نشود، recursion خراب است.

## پیچیدگی‌ها و مقایسه‌ها

| مثال | زمان | حافظه |
| --- | --- | --- |
| Factorial | O(n) | O(n) |
| Binary Search recursive | O(log n) | O(log n) |
| Tree DFS | O(n) | O(h) |
| Fibonacci ساده | O(2^n) | O(n) |

## اشتباهات رایج

- Base case نداشته باشیم.
- مسئله را در هر قدم کوچک‌تر نکنیم.
- Fibonacci ساده را بدون memoization بنویسیم.

## در مصاحبه چطور بگویم؟

> Recursion وقتی خوب است که مسئله به نسخه کوچک‌تر خودش تبدیل شود. حتما base case و حافظه call stack را هم حساب می‌کنم.

## تمرین

1. Factorial را دستی برای ۵ باز کن.
2. Sum یک array را recursive بنویس.
3. Fibonacci ساده و memoized را مقایسه کن.

## پاسخ تمرین‌ها

1. Factorial(5) یعنی 5*Factorial(4)، بعد 4*Factorial(3)، تا Factorial(1). سپس جواب‌ها از پایین برمی‌گردند: 1، 2، 6، 24، 120.
2. برای sum recursive، base case آرایه خالی یا index آخر است. در هر قدم مقدار فعلی را با sum بقیه جمع می‌کنیم. زمان O(n) و حافظه stack برابر O(n) است.
3. Fibonacci ساده محاسبات تکراری زیادی دارد و تقریبا O(2^n) می‌شود. نسخه memoized جواب هر n را ذخیره می‌کند و زمان را به O(n) کاهش می‌دهد.

# فصل ۸: Tree

> ایده خیلی ساده: Tree مثل درخت خانوادگی یا پوشه‌های کامپیوتر است: یک ریشه داریم و شاخه‌ها از آن جدا می‌شوند.

## مثال روزمره

در فایل‌سیستم، یک پوشه اصلی داری. داخلش پوشه‌های دیگر هستند و داخل آن‌ها هم پوشه‌های دیگر.

برای پیدا کردن یک فایل، گاهی باید وارد شاخه‌ها شوی. این ساختار خطی نیست؛ شاخه‌ای است.

Tree برای مدل کردن همین ساختارهای سلسله‌مراتبی است.

## حالا در برنامه‌نویسی یعنی چه؟

Root گره شروع است. Leaf گرهی است که child ندارد.

Binary Tree یعنی هر node حداکثر دو child دارد.

BST یا Binary Search Tree قانونی دارد: چپ کوچک‌تر، راست بزرگ‌تر. اگر متوازن باشد، search می‌تواند O(log n) باشد.

## مثال C#

</div>

```csharp
public sealed class TreeNode
{
    public int Value { get; init; }
    public TreeNode? Left { get; set; }
    public TreeNode? Right { get; set; }
}

public static int MaxDepth(TreeNode? node)
{
    if (node is null) return 0;
    return 1 + Math.Max(MaxDepth(node.Left), MaxDepth(node.Right));
}
```

<div dir="rtl" align="right">

## چیزهایی که باید از این فصل یاد بگیری

- Tree برای داده‌های سلسله‌مراتبی عالی است.
- DFS عمیق می‌رود؛ BFS سطح‌به‌سطح می‌رود.
- هر binary tree لزوما BST نیست.

## پیچیدگی‌ها و مقایسه‌ها

| مفهوم | معنی ساده | مثال |
| --- | --- | --- |
| Root | شروع درخت | پوشه اصلی |
| Leaf | بدون فرزند | فایل یا پوشه خالی |
| Depth | فاصله از root | چند پوشه پایین‌تر |
| Height | بلندترین مسیر تا leaf | عمق زیرشاخه |

## اشتباهات رایج

- Binary Tree و BST را یکی بدانیم.
- درخت خالی را handle نکنیم.
- فرض کنیم BST همیشه O(log n) است؛ اگر کج شود O(n) می‌شود.

## در مصاحبه چطور بگویم؟

> برای traversal درخت، اگر بخواهم عمیق بروم DFS استفاده می‌کنم. اگر سطح‌به‌سطح بخواهم، BFS با Queue مناسب است.

## تمرین

1. Max Depth را برای یک درخت روی کاغذ حساب کن.
2. Inorder traversal را روی BST اجرا کن.
3. Validate BST را با بازه min/max حل کن.

## پاسخ تمرین‌ها

1. Max Depth یعنی بلندترین مسیر از root تا leaf. اگر root خالی باشد 0 است؛ وگرنه 1 + max(depth چپ، depth راست).
2. در BST، inorder یعنی اول چپ، بعد خود node، بعد راست. چون چپ کوچک‌تر و راست بزرگ‌تر است، خروجی مرتب می‌شود.
3. برای Validate BST، هر node باید داخل بازه مجاز باشد. برای فرزند چپ، max برابر مقدار parent می‌شود؛ برای فرزند راست، min برابر مقدار parent می‌شود.

# فصل ۹: Heap و Priority Queue

> ایده خیلی ساده: Heap برای وقتی است که لازم نیست همه چیز کامل مرتب باشد؛ فقط می‌خواهی مهم‌ترین آیتم را سریع برداری.

## مثال روزمره

فرض کن در بیمارستان چند بیمار منتظرند. لازم نیست کل لیست بیماران کامل مرتب باشد، فقط باید هر بار اورژانسی‌ترین بیمار انتخاب شود.

Priority Queue دقیقا همین کار را می‌کند: هر آیتم priority دارد و آیتم با priority مهم‌تر زودتر بیرون می‌آید.

Heap یکی از رایج‌ترین روش‌های ساخت Priority Queue است.

## حالا در برنامه‌نویسی یعنی چه؟

در min-heap، کوچک‌ترین مقدار در root است. در max-heap، بزرگ‌ترین مقدار در root است.

Heap معمولا با array پیاده‌سازی می‌شود. parent و childها با فرمول index حساب می‌شوند.

در .NET از PriorityQueue<TElement, TPriority> استفاده می‌کنیم.

## مثال C#

</div>

```csharp
var queue = new PriorityQueue<string, int>();

queue.Enqueue("normal job", 5);
queue.Enqueue("urgent job", 1);
queue.Enqueue("low priority job", 10);

Console.WriteLine(queue.Dequeue()); // urgent job
```

<div dir="rtl" align="right">

## چیزهایی که باید از این فصل یاد بگیری

- Heap برای search عمومی خوب نیست.
- برای top K، scheduler و Dijkstra عالی است.
- Insert و Remove معمولا O(log n) هستند.

## پیچیدگی‌ها و مقایسه‌ها

| عملیات | پیچیدگی | معنی |
| --- | --- | --- |
| Peek | O(1) | دیدن root |
| Insert | O(log n) | بالا رفتن تا جای درست |
| Remove root | O(log n) | پایین آمدن تا جای درست |
| Build heap | O(n) | heapify |

## اشتباهات رایج

- Heap را با BST اشتباه بگیریم.
- برای top K کل array را sort کنیم.
- priority کوچک‌تر/بزرگ‌تر در .NET را اشتباه بفهمیم.

## در مصاحبه چطور بگویم؟

> اگر فقط کوچک‌ترین/بزرگ‌ترین یا top K را لازم داشته باشم، heap بهتر از sort کامل است چون لازم نیست همه چیز کامل مرتب شود.

## تمرین

1. Top K largest را با heap توضیح بده.
2. PriorityQueue .NET را با چند job تست کن.
3. فرمول parent و childها را برای indexهای ۰ تا ۵ بنویس.

## پاسخ تمرین‌ها

1. برای Top K largest، یک min-heap با اندازه k نگه می‌داریم. اگر عدد جدید از root بزرگ‌تر بود، root را حذف و عدد جدید را اضافه می‌کنیم.
2. در PriorityQueue دات‌نت، priority کوچک‌تر زودتر dequeue می‌شود. مثلا priority 1 قبل از priority 5 بیرون می‌آید.
3. برای index i، فرزند چپ 2i+1، فرزند راست 2i+2 و parent برابر (i-1)/2 است. برای index صفر، فرزندها 1 و 2 هستند.

# فصل ۱۰: Graph

> ایده خیلی ساده: Graph برای مدل کردن رابطه‌هاست: شهرها و جاده‌ها، آدم‌ها و دوستی‌ها، سرویس‌ها و وابستگی‌ها.

## مثال روزمره

فرض کن نقشه شهرها را داری. هر شهر می‌تواند به چند شهر دیگر جاده داشته باشد.

اگر بخواهی از تهران به شیراز برسی، باید مسیرها را بررسی کنی. این دیگر مثل array یا tree ساده نیست؛ رابطه‌ها آزادترند.

Graph برای همین نوع مسئله ساخته شده است.

## حالا در برنامه‌نویسی یعنی چه؟

Graph از vertex و edge تشکیل می‌شود. Vertex یعنی نقطه؛ edge یعنی رابطه یا مسیر.

Adjacency List برای هر vertex لیست همسایه‌ها را نگه می‌دارد.

BFS برای کوتاه‌ترین مسیر در graph بدون وزن مناسب است. DFS برای گشتن عمیق، تشخیص cycle و connected components مفید است.

## مثال C#

</div>

```csharp
public static void Bfs(Dictionary<int, List<int>> graph, int start)
{
    var visited = new HashSet<int> { start };
    var queue = new Queue<int>();
    queue.Enqueue(start);

    while (queue.Count > 0)
    {
        int node = queue.Dequeue();
        Console.WriteLine(node);

        foreach (var next in graph[node])
        {
            if (visited.Add(next))
                queue.Enqueue(next);
        }
    }
}
```

<div dir="rtl" align="right">

## چیزهایی که باید از این فصل یاد بگیری

- در graph باید visited داشته باشی.
- BFS با Queue و DFS با Stack/Recursion انجام می‌شود.
- Graph می‌تواند cycle داشته باشد؛ مثل tree ساده نیست.

## پیچیدگی‌ها و مقایسه‌ها

| روش | کاربرد | پیچیدگی |
| --- | --- | --- |
| Adjacency List | Graph sparse | O(V + E) حافظه |
| Adjacency Matrix | lookup سریع edge | O(V²) حافظه |
| BFS/DFS | گشتن graph | O(V + E) |
| Dijkstra | کوتاه‌ترین مسیر با وزن مثبت | O((V+E) log V) |

## اشتباهات رایج

- visited را فراموش کنیم و وارد حلقه شویم.
- BFS بدون وزن را با Dijkstra قاطی کنیم.
- برای وزن منفی از Dijkstra استفاده کنیم.

## در مصاحبه چطور بگویم؟

> Graph را معمولا با adjacency list نمایش می‌دهم. برای traversal، هر node و edge را حداکثر یک بار می‌بینم، پس BFS/DFS برابر O(V+E) است.

## تمرین

1. Number of Islands را با DFS توضیح بده.
2. Course Schedule را با graph dependency مدل کن.
3. یک graph کوچک بکش و BFS را دستی اجرا کن.

## پاسخ تمرین‌ها

1. در Number of Islands، هر خانه زمین را که ندیده‌ایم شروع DFS/BFS می‌کنیم و همه زمین‌های متصل را visited می‌زنیم. هر شروع جدید یک جزیره است.
2. در Course Schedule، هر درس یک node و هر پیش‌نیاز یک edge است. اگر cycle وجود داشته باشد، گرفتن همه درس‌ها ممکن نیست. با topological sort بررسی می‌کنیم.
3. برای BFS دستی، از node شروع آغاز می‌کنیم، آن را visited می‌زنیم، همسایه‌هایش را در Queue می‌گذاریم، و سطح‌به‌سطح جلو می‌رویم.

# فصل ۱۱: HashMap و HashSet

> ایده خیلی ساده: HashMap مثل دفترچه‌ای است که با کلید، سریع می‌روی سراغ مقدار.

## مثال روزمره

فرض کن در یک شرکت برای هر کارمند یک کد پرسنلی داریم. اگر کد را بدهی، سیستم سریع پرونده همان کارمند را پیدا می‌کند.

لازم نیست بین همه کارمندها بگردی. کلید کمک می‌کند مستقیم‌تر به مقدار برسی.

HashMap همین ایده را در برنامه‌نویسی پیاده می‌کند.

## حالا در برنامه‌نویسی یعنی چه؟

Dictionary<TKey,TValue> در .NET یک HashMap است.

HashSet<T> فقط وجود یا نبودن مقدار را نگه می‌دارد.

عملیات Add، Contains و Get در حالت میانگین O(1) هستند، اما به hash خوب و equality درست وابسته‌اند.

## مثال C#

</div>

```csharp
public static int[] TwoSum(int[] nums, int target)
{
    var indexByValue = new Dictionary<int, int>();

    for (int i = 0; i < nums.Length; i++)
    {
        int need = target - nums[i];

        if (indexByValue.TryGetValue(need, out int j))
            return new[] { j, i };

        indexByValue[nums[i]] = i;
    }

    return Array.Empty<int>();
}
```

<div dir="rtl" align="right">

## چیزهایی که باید از این فصل یاد بگیری

- HashMap برای lookup سریع است.
- HashSet برای حذف تکراری‌ها و contains عالی است.
- حافظه بیشتری مصرف می‌کنیم تا زمان کمتر شود.

## پیچیدگی‌ها و مقایسه‌ها

| ساختار | کاربرد | میانگین زمان |
| --- | --- | --- |
| Dictionary | key -> value | O(1) |
| HashSet | وجود/عدم وجود | O(1) |
| List search | جستجوی ساده | O(n) |

## اشتباهات رایج

- کلید mutable داخل Dictionary بگذاریم و بعد تغییرش دهیم.
- Equals و GetHashCode را ناسازگار پیاده‌سازی کنیم.
- بگوییم همیشه O(1) است، نه میانگین O(1).

## در مصاحبه چطور بگویم؟

> با Dictionary می‌توانم به جای دو حلقه، هر عدد را یک بار ببینم. زمان O(n) می‌شود ولی حافظه O(n) مصرف می‌کنم.

## تمرین

1. Contains Duplicate را با HashSet حل کن.
2. Group Anagrams را با Dictionary توضیح بده.
3. برای یک کلاس سفارشی Equals و GetHashCode را بررسی کن.

## پاسخ تمرین‌ها

1. Contains Duplicate: یک HashSet می‌سازیم. اگر Add برای عددی false شود یعنی قبلا دیده شده و duplicate داریم. زمان O(n)، حافظه O(n).
2. Group Anagrams: برای هر کلمه، حروفش را sort می‌کنیم یا count حروف می‌سازیم و آن را key قرار می‌دهیم. کلمات با key یکسان در یک گروه می‌روند.
3. برای کلاس سفارشی، اگر دو object از نظر Equals برابرند، باید GetHashCode یکسان بدهند. وگرنه Dictionary ممکن است lookup را درست انجام ندهد.

# فصل ۱۲: LRU Cache

> ایده خیلی ساده: LRU یعنی وقتی cache پر شد، چیزی را حذف کن که مدت بیشتری استفاده نشده است.

## مثال روزمره

فرض کن روی میزت فقط جا برای ۳ کتاب داری. هر کتابی را که زیاد استفاده می‌کنی روی میز می‌ماند.

اگر کتاب جدیدی بیاوری و جا نباشد، منطقی است کتابی را برداری که مدت‌هاست سراغش نرفته‌ای.

این همان LRU است: Least Recently Used.

## حالا در برنامه‌نویسی یعنی چه؟

برای Get سریع، Dictionary می‌خواهیم.

برای فهمیدن قدیمی‌ترین استفاده، Doubly Linked List می‌خواهیم.

هر بار آیتمی خوانده یا نوشته می‌شود، به جلوی لیست می‌رود. انتهای لیست قدیمی‌ترین است.

## مثال C#

</div>

```csharp
public sealed class SimpleLru<TKey, TValue> where TKey : notnull
{
    private readonly int _capacity;
    private readonly Dictionary<TKey, LinkedListNode<(TKey Key, TValue Value)>> _map = new();
    private readonly LinkedList<(TKey Key, TValue Value)> _list = new();

    public SimpleLru(int capacity) => _capacity = capacity;

    public bool TryGet(TKey key, out TValue value)
    {
        if (!_map.TryGetValue(key, out var node))
        {
            value = default!;
            return false;
        }

        _list.Remove(node);
        _list.AddFirst(node);
        value = node.Value.Value;
        return true;
    }
}
```

<div dir="rtl" align="right">

## چیزهایی که باید از این فصل یاد بگیری

- LRU ترکیب دو ساختار داده است.
- Dictionary برای پیدا کردن سریع است.
- LinkedList برای ترتیب استفاده اخیر است.

## پیچیدگی‌ها و مقایسه‌ها

| عملیات | پیچیدگی | چرا |
| --- | --- | --- |
| Get | O(1) | Dictionary + move node |
| Put | O(1) | اضافه یا update |
| Evict | O(1) | حذف tail |
| Memory | O(capacity) | map + list |

## اشتباهات رایج

- از Dictionary حذف کنیم ولی node را از لیست حذف نکنیم.
- key را داخل node نگه نداریم و هنگام eviction گیر کنیم.
- thread-safety cache واقعی را فراموش کنیم.

## در مصاحبه چطور بگویم؟

> برای O(1) شدن Get و Put، Dictionary را با Doubly Linked List ترکیب می‌کنم. Dictionary node را پیدا می‌کند و لیست ترتیب استفاده را نگه می‌دارد.

## تمرین

1. LRU با ظرفیت ۲ را روی کاغذ اجرا کن.
2. Put و Get را مرحله‌به‌مرحله بنویس.
3. فکر کن اگر چند thread همزمان cache را بخوانند چه می‌شود.

## پاسخ تمرین‌ها

1. برای ظرفیت 2: Put(1)، Put(2)، Get(1) باعث می‌شود 1 جدیدترین شود. بعد Put(3)، آیتم 2 حذف می‌شود چون دیرتر استفاده شده است.
2. Put اگر key موجود باشد value را update می‌کند و node را جلو می‌آورد. اگر key جدید باشد و ظرفیت پر باشد tail حذف می‌شود، سپس node جدید جلو می‌آید.
3. در چند thread، ممکن است همزمان map و list تغییر کنند و ساختار خراب شود. باید lock، concurrent design یا cache آماده و thread-safe استفاده شود.

# فصل ۱۳: الگوهای مصاحبه

> ایده خیلی ساده: الگوها کمک می‌کنند شکل مسئله را سریع‌تر تشخیص بدهی. مثل جعبه‌ابزار هستند.

## مثال روزمره

اگر پیچ ببینی، پیچ‌گوشتی برمی‌داری. اگر میخ ببینی، چکش. مسئله‌های الگوریتمی هم شکل‌های تکراری دارند.

وقتی substring پیوسته می‌بینی، شاید Sliding Window مناسب باشد. وقتی array مرتب و دو سر مهم‌اند، شاید Two Pointers.

هدف این نیست که جواب حفظ کنی؛ هدف این است که ابزار مناسب را تشخیص بدهی.

## حالا در برنامه‌نویسی یعنی چه؟

Two Pointers معمولا روی array مرتب یا حرکت از دو سمت کاربرد دارد.

Sliding Window برای subarray یا substring پیوسته است.

Backtracking برای ساختن همه حالت‌هاست. DP برای مسئله‌هایی است که زیرمسئله تکراری دارند.

## مثال C#

</div>

```csharp
public static int MaxProfit(int[] prices)
{
    int bestBuy = int.MaxValue;
    int bestProfit = 0;

    foreach (var price in prices)
    {
        bestBuy = Math.Min(bestBuy, price);
        bestProfit = Math.Max(bestProfit, price - bestBuy);
    }

    return bestProfit;
}
```

<div dir="rtl" align="right">

## چیزهایی که باید از این فصل یاد بگیری

- Two Pointers اغلب O(n²) را به O(n) تبدیل می‌کند.
- Sliding Window برای بازه پیوسته است.
- DP یعنی state، transition و base case.

## پیچیدگی‌ها و مقایسه‌ها

| الگو | کی به ذهنم بیاید؟ | مثال |
| --- | --- | --- |
| Two Pointers | array مرتب یا دو سر | Two Sum II |
| Sliding Window | substring/subarray پیوسته | Longest substring |
| Prefix Sum | sum بازه‌ها | Subarray Sum |
| Backtracking | همه حالت‌ها | Subsets |
| DP | زیرمسئله تکراری | Coin Change |

## اشتباهات رایج

- برای هر چیز DP استفاده کنیم.
- Sliding Window را برای داده غیرپیوسته به زور استفاده کنیم.
- در Backtracking انتخاب را undo نکنیم.

## در مصاحبه چطور بگویم؟

> اول شکل مسئله را نگاه می‌کنم. اگر بازه پیوسته باشد sliding window؛ اگر دو سر array مهم باشد two pointers؛ اگر همه حالت‌ها لازم باشد backtracking؛ اگر زیرمسئله تکراری باشد DP.

## تمرین

1. Longest Substring Without Repeating Characters را با sliding window حل کن.
2. Subsets را با backtracking بنویس.
3. Climbing Stairs را با DP توضیح بده.

## پاسخ تمرین‌ها

1. Longest Substring Without Repeating Characters: یک window با left/right داریم و با Dictionary آخرین جای هر کاراکتر را نگه می‌داریم. وقتی تکرار دیدیم left را جلو می‌بریم.
2. Subsets: برای هر عدد دو انتخاب داریم: داخل subset باشد یا نباشد. با backtracking همه انتخاب‌ها را تولید می‌کنیم. تعداد جواب‌ها O(2^n) است.
3. Climbing Stairs: اگر برای رسیدن به پله i می‌توانیم از i-1 یا i-2 بیاییم، پس dp[i] = dp[i-1] + dp[i-2]. پایه‌ها dp[1]=1 و dp[2]=2 هستند.

# فصل ۱۴: روش حل مسئله در مصاحبه

> ایده خیلی ساده: مصاحبه فقط کد نیست؛ باید فکر کردنت را واضح نشان بدهی.

## مثال روزمره

فرض کن مکانیک هستی و ماشین روشن نمی‌شود. اگر مستقیم موتور را باز کنی، شاید اشتباه باشد.

اول سوال می‌پرسی: بنزین دارد؟ باتری سالم است؟ آخرین بار چه اتفاقی افتاد؟

در مصاحبه الگوریتم هم همین است. قبل از کد، مسئله را روشن کن.

## حالا در برنامه‌نویسی یعنی چه؟

اول input و output را دقیق کن. بعد constraints را بپرس.

راه‌حل ساده را بگو، سپس bottleneck را پیدا کن.

بعد با ساختار داده یا الگوی مناسب راه‌حل را بهتر کن. در پایان تست و Big O را بگو.

## مثال C#

```csharp
// قالب ذهنی:

// 5. ساختار داده مناسب انتخاب کن.
```

## چیزهایی که باید از این فصل یاد بگیری

- قبل از کد، مسئله را روشن کن.
- Brute force گفتن بد نیست؛ شروع خوبی است.
- در پایان حتما edge case و complexity بگو.

## پیچیدگی‌ها و مقایسه‌ها

| مرحله | چه بگویم؟ | چرا مهم است؟ |
| --- | --- | --- |
| Clarify | ورودی چقدر است؟ مرتب است؟ duplicate دارد؟ | مسیر حل عوض می‌شود |
| Brute force | راه ساده این است... | نشان می‌دهد مسئله را فهمیدی |
| Optimize | گلوگاه اینجاست... | دلیل انتخاب ابزار مشخص می‌شود |
| Test | حالت خالی، یک آیتم، duplicate | کد قابل اعتمادتر می‌شود |
| Complexity | زمان و حافظه... | جمع‌بندی مهندسی |

## اشتباهات رایج

- بدون سوال پرسیدن شروع به کد زدن کنیم.
- راه‌حل حفظی بدهیم ولی نتوانیم توضیحش دهیم.
- edge caseها را فراموش کنیم.

## در مصاحبه چطور بگویم؟

> من اول مسئله را با مثال کوچک روشن می‌کنم، بعد brute force را می‌گویم، گلوگاه را پیدا می‌کنم، راه بهتر را می‌سازم، تست می‌کنم و در پایان Time و Space را توضیح می‌دهم.

## تمرین

1. برای ۱۰ مسئله، فقط توضیح شفاهی بده و کد نزن.
2. بعد از هر حل، سه edge case بنویس.
3. هر مسئله را با جمله Time و Space تمام کن.

## پاسخ تمرین‌ها

1. برای یک مسئله نمونه، اول input و output را با مثال کوچک مشخص کن. مثلا برای Two Sum بگو ورودی nums و target است و خروجی دو index است.
2. سه edge case رایج: ورودی خالی، فقط یک عنصر، و duplicateها. بسته به مسئله، overflow و null را هم اضافه کن.
3. هر مسئله را این‌طور تمام کن: زمان به این دلیل O(...) است، حافظه به این دلیل O(...) است، و trade-off اصلی این است.
